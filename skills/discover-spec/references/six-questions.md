# 六个逼问 · 打开真需求

借 gstack office-hours 的 YC 六逼问。每个问题三段式:**Ask**(问什么)→ **Push-until**(追问到这个程度才算够)→ **Red-flags**(听到这些要警惕)。
按产品阶段路由:0→1 重点问 1/2/3,1→N 重点问 4/5/6。

## 1 · 需求现实性 Demand Reality
- Ask:有谁现在正为这个问题付钱或付出时间?
- Push-until:能说出具体的人 / 公司 / 场景,不是"大家都需要"。
- Red-flags:"所有人都会用"、"市场很大"、没有任何一个具体用户。

## 2 · 现状替代方案 Status Quo
- Ask:他们今天不用你的东西,是怎么凑合的?
- Push-until:说得出现在的笨办法(Excel / 手动 / 某竞品 / 忍着)。
- Red-flags:"现在没有解决方案"(几乎总是错的,只是你没找到)。

## 3 · 绝望级具体性 Desperate Specificity
- Ask:谁对这个问题已经痛到愿意马上付钱?
- Push-until:锁定一个最痛的细分人群 + 一个最痛的具体场景。
- Red-flags:泛泛的"中小企业 / 年轻人 / 创作者"。

## 4 · 最窄切入 Narrowest Wedge
- Ask:能砍到多小,还依然有人要?
- Push-until:一个一句话能说清、一周能做出的最小切口。
- Red-flags:"平台"、"生态"、"全家桶"、什么都想做。

## 5 · 观察与意外 Observation & Surprise
- Ask:你亲眼看到的、和直觉相反的用户行为是什么?
- Push-until:一个具体的、你没预料到的真实观察。
- Red-flags:全是推测 / 二手转述 / 没真见过用户。

## 6 · 未来契合 Future-Fit
- Ask:为什么是现在?什么变了让它现在成立?
- Push-until:一个具体的趋势 / 技术 / 行为变化。
- Red-flags:"一直都需要"(那为什么之前没人做成?)。

## 反谄媚(贯穿全程)
禁说"这想法很棒 / 有意思"。每个回答都要表态:哪里站不住、哪里有风险、哪个假设最脆弱。

## 逆向版映射(Mode B,无真人可问时)

被当 subagent 自动跑、或纯逆向时,没有真人回答六逼问。规则:把每个逼问转成 `[ASSUMPTION]` 自答 + 标来源,写进 spec 的指定段落,**不要跳过**。

| 逼问 | 逆向时怎么答 | 写进 spec 哪段 |
| --- | --- | --- |
| 谁用 / 需求现实性 | 从产品定位 + 竞品反推目标用户 | 背景 / 问题 |
| 现状替代方案 | 竞品就是现状替代 | ASSUMPTION |
| 为什么是现在 | 从市场 / 技术变化推 | 背景 |
| 最窄切入 | MVP 复刻范围 | 复刻范围段 |
| 明确不做 | 产品没做的 + 你不抄的 | out-of-scope |
| 观察与意外 | 竞品分析里的 commonGap | 差异化点 |

每条标 `[ASSUMPTION: … | 来源: …]`(如 SERP 实测 / teardown §x),不伪装成事实。
