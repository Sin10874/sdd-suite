# Mode B 产物模板 · 复刻范围 / 差异化点

Mode B(复刻已验证产品)的 spec 必须比默认多两段:`## 复刻范围` 和 `## 差异化点`。
缺这两段,spec 不算完成(validate 会拦)。

## ## 复刻范围

一张表锁定"抄什么、抄到什么程度":

| 复刻项 | 抄什么(到什么程度) | 对应 AC |
| --- | --- | --- |
| 核心功能 A | 抄交互与算法口径,不抄视觉 | AC-001, AC-002 |
| 数据层 | 抄数据结构,自建数据源 | AC-003 |
| … | … | … |

配一句 MVP 边界:复刻的最小切片是什么(别一上来抄全集)。

## ## 差异化点

**硬规则:差异化点至少有 1 条落地条目,或明确标 `[ASSUMPTION: ...]`;不得整段全是 `[NEEDS CLARIFICATION]`。**

正面复刻必死——价值全在"换什么 niche、凭什么赢"。这段空白 = 立项理由悬空 = 过不了下一阶段的门。

每条差异化写成:

- **打哪个 gap**:(来自竞品分析的 commonGap)
- **怎么打**:具体做法
- **护城河**:为什么别人不容易跟

## 竞品洞察 → 差异化点(接线规则)

当有竞品分析传入(`commonGaps` / `differentiation` 列表),**逐条映射,一条都不能丢**:

- 每条 commonGap → 要么变成一条差异化点(我们来填这个坑),要么进 out-of-scope(明确不碰)。
- 每条 differentiation 机会 → 变成一条差异化条目,或一条 `[ASSUMPTION: ...]`。
- 映射完在差异化段末尾留一行:`> 竞品洞察已逐条消化,未采纳的已列入 out-of-scope`。

没有这个动作,辛苦做的竞品研究会被丢掉——这正是 PaycheckCity dogfood 暴露的洞。
